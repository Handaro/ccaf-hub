D1: Agentic Architecture & Orchestration
Domain 1 · 27% of exam · ~3 min narration
0:003:10
Domain one covers agentic architecture and orchestration, the largest domain at twenty-seven percent. The core mental model is the agentic loop. Claude sends a request, the system checks the stop reason, executes any tool calls, returns results, and repeats until end turn. Know this loop cold. It underlies sixty-five percent of the exam. The critical exam trap: when a workflow skips mandatory checks, use a programmatic hook, not a system prompt. Hooks are deterministic. System prompts are probabilistic.

D2: Claude Code Configuration & Workflows
Domain 2 · 20% of exam
0:002:20
Domain two covers Claude Code configuration. The most important file is CLAUDE.md, which lives at the repo root and loads automatically every session. For granular control, use the .claude/rules/ directory with YAML path-scoping. Rules only activate for matching file patterns. Adding context:fork in a skill frontmatter runs it isolated, preventing verbose output from polluting your main session.

D4: Tool Design & MCP Integration
Domain 4 · 18% of exam
0:002:00
Domain four covers the Model Context Protocol. Think of MCP as USB-C for AI integrations: one standard interface for all connections. MCP has three core primitives: Tools for actions, Resources for data, and Prompts for templates. Claude selects tools purely by reading descriptions. If the wrong tool is chosen, fix the description.

D5: Context Management & Reliability
Domain 5 · 15% of exam
0:002:00
Domain five is about context management and reliability. The context window is a shared token budget. In long-running agents, history grows every turn. Progressive summarization compresses old turns into summaries. Prompt caching reduces latency by eighty percent. Circuit breakers prevent infinite loops. Provenance tracks which agent produced each fact.
